# คู่มือการติดตั้งและใช้งาน RD Checker (ละเอียดมาก / ภาษาไทย)

เอกสารนี้ตั้งใจเขียนให้ “คนที่ไม่ค่อยถนัดคอมพิวเตอร์” ก็ทำตามได้ทีละขั้น โดยไม่ต้องเขียนโค้ดและไม่ต้องติดตั้งโปรแกรมเพิ่ม

---

## RD Checker คืออะไร (อ่าน 30 วินาที)
RD Checker เป็นส่วนเสริม (Extension) ของ Google Chrome เอาไว้ช่วย “ดึงข้อมูลจากไฟล์” แล้ว “ตรวจ/เทียบผล” อัตโนมัติ เช่น

1. ดึงข้อมูลจากไฟล์ใบงาน (PDF)  
   - RD Code  
   - วันที่ดำเนินการเสร็จ (Completion Date)  
   - Software Version

2. ดึงข้อมูลจาก Log Switch (GUI หรือ CLI)  
   - เวลา `show clock` (Last Time)  
   - เวลาห่างระหว่างเวลา “จุดที่ 2 และ 3” ของ `show clock`  
   - ตรวจ `show interface counters errors` (โหมด GUI)  
   - ตรวจ `show interface | I CRC` (โหมด CLI)  
   - ตรวจจับคำสั่ง/Log ที่มีคำว่า `clear counters` / `%CLEAR-5-COUNTERS`

3. กด “Compare Data” เพื่อสรุปผลรวม
   - เทียบ Version (PDF vs Log Switch)  
   - เทียบเวลา (Last log time ต้องไม่เกินวันที่ดำเนินเสร็จในใบงาน)  
   - สรุปผล CRC / Interface errors และการ clear counters

## 1) เตรียมของก่อนเริ่ม (เช็คลิสต์)
ก่อนเริ่ม ให้เตรียมสิ่งเหล่านี้ให้พร้อม

1. คอมพิวเตอร์ Windows 10/11
2. โปรแกรม Google Chrome (แนะนำให้เป็นเวอร์ชันใหม่ ๆ)
3. โฟลเดอร์โปรเจกต์ `rd-checker` (โฟลเดอร์ที่มีไฟล์ `manifest.json` อยู่ข้างใน)
4. ไฟล์ที่ต้องใช้ตรวจ อย่างน้อย 2 อย่างนี้
   - ไฟล์ใบงาน (PDF) “รายงานการบำรุงรักษา…”
   - ไฟล์ Log Switch (GUI หรือ CLI) อาจเป็น `PDF` หรือ `HTML` หรือ `TXT` หรือ URL จากเว็บ

ถ้าคุณได้รับโปรเจกต์มาเป็นไฟล์ `.zip` (เช่น `rd-checker.zip`):
1. ไปที่โฟลเดอร์ที่เก็บไฟล์ zip (ส่วนใหญ่จะอยู่ที่ Downloads)
2. คลิกขวาที่ไฟล์ `.zip`
3. เลือก “Extract All…” / “แตกไฟล์ทั้งหมด…”
4. กด “Extract”
5. จะได้โฟลเดอร์ `rd-checker` (ข้างในต้องมี `manifest.json`)

หมายเหตุ: ถ้า Log อยู่บนเว็บที่ต้องล็อกอิน/มี VPN ให้ล็อกอิน/ต่อ VPN ให้เรียบร้อยก่อน

## 2) ติดตั้ง Extension (ทำครั้งเดียว)
ทำตามทีละข้อแบบนี้

### 2.1 เปิดหน้า Extensions ของ Chrome
1. เปิด Google Chrome
2. คลิกที่ช่องด้านบนสุดที่พิมพ์เว็บ (Address bar)
3. พิมพ์ `chrome://extensions` แล้วกด Enter

### 2.2 เปิดโหมดนักพัฒนา (Developer mode)
1. มองหาคำว่า “Developer mode” ที่มุมขวาบนของหน้า
2. กดสวิตช์ให้เป็น “เปิด” (On)

### 2.3 โหลดโฟลเดอร์ RD Checker
1. กดปุ่ม “Load unpacked”
2. จะมีหน้าต่างเลือกโฟลเดอร์เด้งขึ้นมา
3. เลือกโฟลเดอร์ `rd-checker`
   - สำคัญ: ต้องเป็นโฟลเดอร์ที่ข้างในเห็นไฟล์ `manifest.json`
   - ตัวอย่างตำแหน่งในเครื่องของผู้พัฒนา: `D:\mycode\rd-checker`
4. กด “Select Folder” / “เลือกโฟลเดอร์”
5. ถ้าสำเร็จ จะเห็น “RD Checker” โผล่ในรายการ Extensions

ข้อควรระวัง (สำคัญ):
หลังจากกด “Load unpacked” แล้ว **อย่าย้าย/เปลี่ยนชื่อ/ลบ** โฟลเดอร์ `rd-checker` เด็ดขาด
เพราะ Chrome จะอ้างอิงไฟล์จากโฟลเดอร์นี้ตลอด ถ้าย้ายที่เก็บ Extension จะพังและต้อง Load ใหม่

### 2.4 ปักหมุด (Pin) เพื่อกดใช้ง่าย
1. มองที่มุมขวาบนของ Chrome (ใกล้ ๆ ช่อง URL) จะมีไอคอนรูปจิ๊กซอว์ (Extensions)
2. กดไอคอนจิ๊กซอว์
3. หา “RD Checker” แล้วกดรูปหมุด (Pin)
4. ต่อไปจะเห็นไอคอน RD Checker อยู่บนแถบบนตลอด กดใช้งานได้ทันที

## 3) ตั้งค่า “ให้เปิดไฟล์ในเครื่องได้” (สำคัญมาก)
ถ้าคุณจะใช้ไฟล์ที่อยู่ในคอมพิวเตอร์ (เช่น PDF/HTML ที่ดาวน์โหลดมา) ต้องเปิดสิทธิ์นี้ก่อน ไม่งั้นจะอ่านไฟล์ไม่ได้

1. ไปที่ `chrome://extensions`
2. หา “RD Checker”
3. กดปุ่ม “Details” (รายละเอียด)
4. มองหาสวิตช์ “Allow access to file URLs”
5. เปิดสวิตช์นี้ให้เป็น “On”

## 4) วิธีใส่ “พาธไฟล์” หรือ “ลิงก์” ให้ถูกต้อง (แบบคนไม่ถนัดคอม)
ใน RD Checker จะมีช่องให้ใส่ “Path ไฟล์ หรือ URL” 2 จุด คือ
1. ช่องบนสุด: สำหรับ “ไฟล์ใบงาน / PDF”
2. ช่อง “Log Switch GUI / CLI”: สำหรับ “ไฟล์/เว็บ Log Switch”

### 4.1 ถ้าไฟล์อยู่ในเครื่องเรา (Windows) — วิธีหาพาธแบบง่าย
1. เปิดโฟลเดอร์ที่มีไฟล์ (เช่น Downloads)
2. คลิกไฟล์ที่ต้องการ 1 ครั้ง (ยังไม่ต้องเปิดไฟล์)
3. กด `Shift` ค้าง แล้วคลิกขวาที่ไฟล์
4. เลือก “Copy as path” / “คัดลอกเป็นเส้นทาง”
5. เอาไปวางในช่องของ RD Checker ได้เลย

ตัวอย่างพาธที่คัดลอกได้ (อาจมีเครื่องหมายคำพูด `"` ติดมาด้วย)

```
"C:\Users\yourname\Downloads\FDO25040LT0test11.pdf"
```

หมายเหตุ: RD Checker รองรับการวางพาธแบบ Windows ได้แล้ว (ไม่จำเป็นต้องแปลงเป็น `file:///...` เอง)
ถ้าพาธมีเครื่องหมายคำพูดครอบอยู่ (เช่น `"C:\...\file.pdf"`) ถือว่าปกติ ระบบจะตัดออกให้เอง

### 4.2 ถ้าเป็นลิงก์เว็บ (URL) — วิธีเช็คว่าถูกไหม
URL มักขึ้นต้นด้วย `https://`

1. ลองเปิด URL นั้นใน Chrome ก่อน (เพื่อให้แน่ใจว่าเข้าถึงได้จริง)
2. ถ้าเปิดได้และเห็นไฟล์/หน้าเว็บตามต้องการ ค่อยคัดลอก URL มาใส่ใน RD Checker

ถ้า URL ต้องล็อกอิน: ให้ล็อกอินใน Chrome ให้เรียบร้อยก่อน แล้วค่อยกด Extract

## 5) วิธีใช้งานจริง (ทำตามลำดับนี้ทุกครั้ง)
การใช้งานปกติจะมี 3 ขั้นใหญ่ ๆ:

1. ดึงข้อมูลจากใบงาน (PDF)  
2. ดึงข้อมูลจาก Log Switch (GUI หรือ CLI)  
3. กด Compare เพื่อสรุปผล

### 5.1 ดึงข้อมูลจาก “ไฟล์ใบงาน / PDF”
1. กดไอคอน RD Checker ที่ปักหมุดไว้
2. ในหัวข้อ “📄 รายงานการบำรุงรักษาอุปกรณ์เครือข่ายคอมพิวเตอร์”
3. วางพาธไฟล์ PDF หรือ URL ของ PDF ลงในช่อง
4. กดปุ่ม “Extract PDF Data”
5. รอสักครู่ แล้วคลิก “ไฟล์ใบงาน / PDF Results (คลิกเพื่อดู)”
6. ตรวจสอบว่ามีข้อมูลขึ้นหรือไม่
   - RD Code
   - วันที่ดำเนินเสร็จ(ว/ด/ป)
   - Software Version

ถ้าขึ้น “Not found”:
1. เปิด “📄 Text Preview (Debug)”
2. ดูว่ามีข้อความ/รูปแบบข้อมูลอยู่ในไฟล์จริงไหม
3. บางไฟล์เป็นรูปภาพสแกน (ไม่มีตัวอักษรจริง) ระบบจะหาไม่เจอ (ต้องใช้ OCR ซึ่งตอนนี้ยังไม่รองรับ)

### 5.2 ดึงข้อมูลจาก “Log Switch” (เลือก GUI หรือ CLI ให้ถูก)
ก่อนกด Extract ให้เลือกโหมดให้ตรงกับไฟล์ Log ที่มี

**เลือก GUI** ถ้า Log มีลักษณะประมาณนี้
1. มีคำสั่ง `show interface counters errors`
2. มีตารางที่มีคอลัมน์ เช่น `Port Align-Err CRC-Err ...`

**เลือก CLI** ถ้า Log มีลักษณะประมาณนี้
1. มีคำสั่ง `show clock` (หรือ `sh clock`)
2. มีคำสั่ง `show interface | I CRC` (หรือ `sh int | i CRC`)

ขั้นตอน
1. ในหัวข้อ “Log Switch GUI / CLI”
2. เลือก `GUI` หรือ `CLI`
3. วาง URL หรือพาธไฟล์ Log ลงในช่อง
   - ถ้าเป็นไฟล์ในเครื่อง แนะนำให้วางพาธแบบ Windows ได้เลย เช่น `C:\...\log.pdf` หรือ `C:\...\log.html`
4. กดปุ่ม “Extract Website Data”
5. รอสักครู่ แล้วคลิก “Log Switch GUI / CLI Results (คลิกเพื่อดู)”

หมายเหตุ:
- ถ้าคุณใส่เป็น URL เว็บ ระบบอาจเปิดแท็บใหม่ “ชั่วคราว” เพื่อดึงข้อมูล แล้วปิดแท็บเอง (เป็นปกติ)
- ถ้าคุณใส่เป็นไฟล์ในเครื่อง ระบบจะแจ้ง “โหมดทดสอบ: ใช้ไฟล์จากเครื่อง” (เป็นปกติ)

### 5.3 อ่านความหมายผลใน “Log Switch GUI / CLI Results”
ผลที่เห็นจะมีหัวข้อสำคัญดังนี้

1. `Last Time`  
   - คือเวลาสุดท้ายที่เจอจาก `show clock`
2. `เวลาระหว่าง 2 กับ 3`  
   - ระบบจะใช้ `show clock` จุดที่ 2 และ 3 มาคำนวณว่า “ห่างกันกี่นาที”
   - เงื่อนไขปัจจุบัน: ต้องห่างกัน **อย่างน้อย 5 นาที**
3. `Interface counters`  
   - โหมด GUI: ตรวจ `show interface counters errors`  
     - ถ้าแถวข้อมูลมีค่าหลัง port เป็น `--` ทั้งหมด = ปกติ  
     - ถ้ามีตัวเลข/ตัวอักษร (เช่น `98`, `87`, `a`, `ห`) = เตือน และจะแสดงว่าอยู่ port ไหน
   - โหมด CLI: ตรวจ `show interface | I CRC`  
     - ถ้าค่าทุกบรรทัดเป็น `0 input errors, 0 CRC, ...` = ปกติ  
     - ถ้ามีค่าไม่เป็น 0 = เตือน และจะแสดงบรรทัดที่มีปัญหา
4. `Detect "Clear counters"`  
   - ถ้าเจอคำสั่ง `clear counters` จะขึ้นเตือน (ต้องแก้ไข/เก็บ log ใหม่ตามขั้นตอนหน้างาน)
5. `Detect "Clear"`  
   - ถ้าเจอ Log `%CLEAR-5-COUNTERS` จะขึ้นเตือน

ถ้าอยากดูละเอียด ให้เปิด “🔍 Debug Info (Website)” แล้วดูบรรทัดที่ระบบจับได้

### 5.4 กด Compare Data เพื่อสรุปผลรวม
1. หลังจากดึงข้อมูลจาก PDF และ Log Switch แล้ว
2. กดปุ่ม “Compare Data”
3. รอสักครู่ จะมีหัวข้อสรุป เช่น
   - Version (เทียบ PDF vs Log Switch)
   - เวลาสุดท้ายในการเก็บ log เทียบกับเวลาที่ดำเนินการเสร็จ
   - เวลาของ Log Switch ระหว่างเวลาที่ 2 กับ 3
   - CRC / interface errors
   - Clear counters / Clear

## 6) ปัญหาที่พบบ่อย + วิธีแก้

### ไม่เห็นปุ่ม Developer mode หรือ Load unpacked
1. ให้ไปที่ `chrome://extensions` ให้ถูกหน้า
2. ถ้าหน้าจอเล็ก/ซูมใหญ่ ให้ลองกด `Ctrl` + `-` เพื่อลดซูม แล้วมองที่มุมขวาบนใหม่อีกครั้ง
3. บางเครื่อง Chrome เป็นภาษาไทย อาจเขียนว่า “โหมดนักพัฒนา”

### กด Load unpacked แล้วขึ้น Error เรื่อง manifest
สาเหตุส่วนใหญ่คือ “เลือกโฟลเดอร์ผิด”
1. กลับไปกด Load unpacked ใหม่
2. เลือกโฟลเดอร์ที่ข้างในเห็นไฟล์ `manifest.json` เท่านั้น

### เปิด RD Checker แล้วไม่ขึ้นหน้าต่าง
1. ไปที่ `chrome://extensions`
2. ดูว่า RD Checker เปิดอยู่หรือไม่ (สวิตช์ต้องเป็น On)
3. ถ้ายังไม่ได้ ให้ลองปิด/เปิด Chrome ใหม่
4. ถ้ายังไม่ขึ้น ให้กดรูปไอคอนจิ๊กซอว์ (Extensions) แล้วเลือก RD Checker จากในรายการ

### ใส่พาธไฟล์แล้ว Extract ไม่ได้ / ขึ้น Error
1. ตรวจว่าพาธไฟล์ถูกต้องและไฟล์มีอยู่จริง
2. ไปที่ `chrome://extensions` → RD Checker → Details
3. เปิด “Allow access to file URLs” ให้เป็น On
4. ถ้าไฟล์ชื่อมีช่องว่าง/ภาษาไทยแปลก ๆ แล้วอ่านไม่เจอ ให้ลองเปลี่ยนชื่อไฟล์เป็นอังกฤษง่าย ๆ แล้วลองใหม่
5. ถ้าเป็น URL เว็บแล้วขึ้น Error/CORS ให้ลอง “ดาวน์โหลดไฟล์มาไว้ในเครื่อง” แล้วใช้พาธไฟล์แทน

### ขึ้น Not found ทั้งหมด
1. ไฟล์อาจเป็น “สแกนรูปภาพ” ไม่มีข้อความจริง ระบบจึงหาไม่เจอ
2. ลองเปิด “Text Preview (Debug)” เพื่อดูว่ามีข้อความออกมาหรือไม่
3. ถ้าเป็นไฟล์สแกน ให้ขอไฟล์แบบที่เลือกข้อความได้ หรือทำ OCR ก่อน (ตอนนี้ Extension ยังไม่ทำ OCR ให้)

### กด Compare Data แล้วผลไม่ครบ / เหมือนไม่มีข้อมูล
1. ต้องกด “Extract PDF Data” ให้สำเร็จก่อน (ให้เห็น RD Code / วันที่ / Version ใน PDF Results)
2. ต้องกด “Extract Website Data” ให้สำเร็จก่อน (ให้เห็น Last Time / Version / Interface counters ใน Results)
3. แล้วค่อยกด “Compare Data” อีกครั้ง

### ผล show clock ไม่เจอ หรือขึ้นว่า “โชว์ไม่ครบลักษณะ”
1. ตรวจในไฟล์ Log ว่ามีคำสั่ง `show clock` หรือ `sh clock` จริงหรือไม่
2. โดยปกติควรมีรูปแบบประมาณนี้:

```text
<hostname># show clock
16:15:41.673160 +07 Mon Nov 24 2025
```

3. ถ้า Log ถูกตัด/จัดรูปแบบแปลกมาก ให้ดูใน “🔍 Debug Info (Website)” แล้วส่งให้คนช่วยวิเคราะห์

### เจอผล `Gi9/9 -- 12 TO HQ --` ทั้งที่ไม่ได้เกี่ยวกับตาราง (กรณี GUI)
เวอร์ชันล่าสุดของ RD Checker จะสแกนเฉพาะส่วนผลลัพธ์หลังคำสั่ง `show interface counters errors` เพื่อลด false positive จาก running-config/description

## 7) (ทางเลือก) ส่งข้อมูลให้ช่วยวิเคราะห์
ถ้าต้องการส่งให้คนช่วยดู (โดยไม่ส่งไฟล์จริง) ให้ทำแบบนี้

1. กด RD Checker
2. Extract PDF และ/หรือ Extract Website Data
3. เปิด “📄 Text Preview (Debug)” หรือ “🔍 Debug Info (Website)”
4. คัดลอกข้อความในกล่อง แล้วส่งให้ผู้ช่วยตรวจสอบ
